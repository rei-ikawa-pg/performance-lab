# Performance Lab — 仕様書

## 1. アプリケーション概要

任意のURLを入力するとCore Web Vitals（LCP, CLS, INP, FCP, TTFB）をPageSpeed Insights API経由で計測し、スコア可視化・改善提案レポートを提供するWebアプリケーション。

## 2. ユーザーフロー

### 2-1. 未認証ユーザー

1. トップページにアクセス → Google認証を促すUI表示
2. Googleアカウントでログイン

### 2-2. 認証済みユーザー（計測フロー）

1. URL入力フォームにURLを入力
2. 「計測」ボタンをクリック
3. ローディング表示（PageSpeed Insights APIは10〜30秒程度かかる）
4. 計測結果ページに遷移
   - スコアカード（LCP, CLS, INP, FCP, TTFB）
   - D3.jsゲージチャート
   - 改善提案レポート一覧
5. 計測結果はSupabaseに自動保存（Should機能）

### 2-3. ダッシュボード（Should）

1. 登録済みURLの最新スコア一覧表示
2. 過去の計測結果を時系列チャートで比較

## 3. 画面一覧

| 画面           | パス           | 認証 | 説明                                                   |
| -------------- | -------------- | ---- | ------------------------------------------------------ |
| トップページ   | `/`            | 不要 | URL入力フォーム + サービス説明。未認証時はログイン誘導 |
| 計測結果       | `/report/[id]` | 必要 | スコアカード + ゲージチャート + 改善提案レポート       |
| ダッシュボード | `/dashboard`   | 必要 | 登録URL一覧 + 最新スコア + 履歴推移チャート            |

## 4. API設計

### 4-1. 認証API

- `GET/POST /api/auth/[...nextauth]` — NextAuth.js v5 のハンドラ

### 4-2. 計測API

- `POST /api/measure`
  - リクエスト: `{ url: string, strategy?: "mobile" | "desktop" }`
  - レスポンス: 正規化された計測結果JSON
  - 認証: 必須（NextAuth session）
  - Rate Limit: 1ユーザーあたり10リクエスト/時

### 4-3. 計測履歴API（Should）

- `GET /api/measurements` — ログインユーザーの計測履歴一覧
- `GET /api/measurements/[id]` — 計測結果詳細

## 5. データモデル

### users（NextAuth管理）

NextAuth.jsのデフォルトスキーマを使用（accounts, sessions, users, verification_tokens）

### measurements

| カラム            | 型                   | 説明                            |
| ----------------- | -------------------- | ------------------------------- |
| id                | uuid (PK)            |                                 |
| user_id           | text (FK → users.id) |                                 |
| url               | text                 | 計測対象URL                     |
| strategy          | text                 | "mobile" or "desktop"           |
| performance_score | integer              | 総合スコア (0-100)              |
| lcp               | float                | Largest Contentful Paint (ms)   |
| cls               | float                | Cumulative Layout Shift         |
| inp               | float                | Interaction to Next Paint (ms)  |
| fcp               | float                | First Contentful Paint (ms)     |
| ttfb              | float                | Time to First Byte (ms)         |
| audits            | jsonb                | 改善提案の生データ              |
| raw_response      | jsonb                | APIレスポンス全体（デバッグ用） |
| created_at        | timestamptz          |                                 |

### urls（Should: ダッシュボード用）

| カラム     | 型                   | 説明        |
| ---------- | -------------------- | ----------- |
| id         | uuid (PK)            |             |
| user_id    | text (FK → users.id) |             |
| url        | text                 | 監視対象URL |
| label      | text                 | 表示名      |
| created_at | timestamptz          |             |

## 6. スコア判定基準（Google公式準拠）

| メトリクス | Good     | Needs Improvement | Poor     |
| ---------- | -------- | ----------------- | -------- |
| LCP        | ≤ 2500ms | ≤ 4000ms          | > 4000ms |
| CLS        | ≤ 0.1    | ≤ 0.25            | > 0.25   |
| INP        | ≤ 200ms  | ≤ 500ms           | > 500ms  |
| FCP        | ≤ 1800ms | ≤ 3000ms          | > 3000ms |
| TTFB       | ≤ 800ms  | ≤ 1800ms          | > 1800ms |

カラーコード: Good = `#0CCE6B`, Needs Improvement = `#FFA400`, Poor = `#FF4E42`

## 7. 非機能要件の詳細

### パフォーマンス

- 自アプリのLighthouseスコア90+
- D3.jsはdynamic importで遅延ロード
- Server Componentsを最大限活用

### セキュリティ

- PageSpeed Insights APIキーはサーバーサイドのみ（`process.env`）
- Rate Limiting: Upstash Redis または in-memory Map
- 認証必須のルートはNextAuth middlewareで保護

### アクセシビリティ

- カラーコントラスト比 4.5:1 以上
- チャートにaria-label付与
- キーボード操作対応
